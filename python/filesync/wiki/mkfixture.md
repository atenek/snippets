# mkfixture — описание и руководство

Генератор **детерминированного тестового дерева** и **оракула ожиданий** для автотеста `get`. Запускается **прямо на удалённом хосте**, автономно (только stdlib).

> Полная спецификация (каталог кейсов, субъекты, матрицы прав) — в [ТЗ mkfixture](../todo/done/mkfixture.md).

---

## Что делает

Создаёт под `--root` дерево объектов с разными типами, правами и владельцами (обычные файлы, директории, симлинки, fifo/socket/устройства, жёсткие ссылки, краевые случаи) и пишет рядом оракул:

```
{root}/
├── tree/                      # тестовое дерево (объекты кейсов)
└── _oracle/
    ├── expectations.json      # эталон ожиданий по каждому объекту + фактические uid/gid субъектов
    └── filelist.txt           # входной список путей для getfile
```

Контент и mtime детерминированы (фиксированы), поэтому MD5 и время предсказуемы. Фактические `uid`/`gid` тестовых субъектов записываются в оракул (различаются между дистрибутивами).

---

## Предпосылки

- `python3` (3.10+) на удалённом хосте;
- запуск под **`sudo`/root** — для создания объектов с чужими владельцами и спец-файлов (устройства).

Без root скрипт работает, но **привилегированные кейсы пропускаются** (помечаются `skipped` в оракуле) — пригодно для беглой проверки, не для полного набора.

---

## Запуск

Скрипт автономный — его копируют на удалённый хост и запускают там (автотест делает это сам).

```bash
# создать фикстуру (полный набор — под sudo)
sudo python3 mkfixture.py --root /tmp/filesync_fixture

# удалить фикстуру (и созданные --create-users учётки)
sudo python3 mkfixture.py --root /tmp/filesync_fixture --clean

# беглая проверка без root (привилегированные кейсы пропущены)
python3 mkfixture.py --root /tmp/filesync_fixture
```

### Ключи

| Ключ | По умолчанию | Описание |
|---|---|---|
| `--root` | `/tmp/filesync_fixture` | корень дерева (изолированный, безопасный для очистки) |
| `--clean` | — | удалить `--root` (и `--create-users` учётки) и выйти |
| `--seed` | `0` | зерно детерминированного контента |
| `--no-privileged` | off | не создавать кейсы, требующие root |
| `--create-users` | off | создать выделенные `fsynctest` user/group вместо `nobody`/`nogroup` |

---

## Тестовые субъекты

4 пользователя и 4 группы (см. [ТЗ §5](../todo/done/mkfixture.md)): `SELF` (вызывающий по `$SUDO_USER`), `ROOT`, `OTHER` (`nobody`/`fsynctest`), `ORPHAN` (uid/gid без имени → `UNKNOWN`). Существующие учётки используются по умолчанию; членство SSH-пользователя не изменяется.

---

## Идемпотентность

- Повторный запуск пересоздаёт дерево «с нуля».
- `--clean` полностью удаляет `--root`; при `--create-users` — также удаляет `fsynctest` (передайте тот же флаг к `--clean`).
- Скрипт не трогает ничего вне `--root`.

---

## Связь с автотестом

`mkfixture` используется автотестом [test_getfile](test_getfile.md): тот копирует скрипт на удалённый хост, запускает под sudo, забирает оракул и сверяет с ним результат `get`. Вручную mkfixture обычно запускать не нужно — только для отладки фикстуры.
