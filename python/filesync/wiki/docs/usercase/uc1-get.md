# UC1 — get: забрать набор файлов с хоста на локальную машину

`HOST1 → HOST_LOCAL1`

## Действия пользователя

1. Составить локальный список путей (по одному на строку; `#` — комментарий):
   ```
   # files.txt
   /etc/nginx/nginx.conf
   /etc/hosts
   ```
2. Запустить get:
   ```bash
   python -m filesync get files.txt 192.168.234.129 --prefix clusterA --suffix message -p ./backups/
   ```

## Ожидаемый результат (кратко)

Создаётся сессия `./backups/clusterA__192.168.234.129__<дата>__message/`
(без `/` — `clusterA__<host>__<дата>__message` одной папкой; со `/` в prefix/suffix — дерево):
- `files/` — raw-files-local (browsable-зеркало);
- `192.168.234.129__<дата>__message.tar` — faithful payload (RO);
- `restore.txt` — editable override атрибутов (для put);
- `metadata/manifest.json|.csv|session.json` — метаданные (RO).

В консоли — сводка: сколько забрано / metadata-only / нет прав / не найдено и
результат MD5-верификации. Код возврата `0` (без пропусков) или `2` (есть
пропуски/расхождения).

## Справочно: ключи и значения по умолчанию

| Ключ | По умолчанию (если не указан) | Описание |
|---|---|---|
| `filelist` (поз.) | — (обязателен) | локальный файл со списком удалённых путей |
| `hostname` (поз.) | — (обязателен) | хост из `~/.ssh/config` (здесь `192.168.234.129`) |
| `--prefix` | `""` | префикс имени; со `/` — каталоги перед сессией (дерево), без `/` — склейка |
| `--suffix` | `""` | суффикс имени; со `/` — вложенные каталоги после сессии, без `/` — склейка |
| `-p`, `--path` | `../backup_files/` | базовая папка для сессий |

Минимальный вызов (всё по умолчанию): `python -m filesync get files.txt 192.168.234.129`
→ сессия `../backup_files/192.168.234.129__<дата>/` без метки.

> Требуется passwordless sudo на хосте для `find/stat/tar/md5sum/readlink`.
