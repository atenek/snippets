# UC5 — get с правилами исключения (ignore)

`HOST → HOST_LOCAL` с фильтром «шума» по правилам, не перечисляя все файлы.

Полная спецификация — [../../todo/done/exclude_rules.md](../../todo/done/exclude_rules.md).

## Действия пользователя

1. Создать ignore-файл (по умолчанию `./.syncignore`) в стиле `.gitignore`:
   ```
   # .syncignore
   venv/ .venv/ node_modules/
   __pycache__/ *.pyc
   .idea/ .vscode/
   *.log log/ run/ runs/ tmp/
   ```
2. В filelist указать **корневые папки** (и при желании отдельные файлы):
   ```
   /srv/app
   ```
3. Запустить get с `--exclude`:
   ```bash
   # правила из ./.syncignore
   python -m filesync get files.txt 192.168.234.129 --exclude --suffix proj -p ./backups/

   # или явный путь к файлу правил
   python -m filesync get files.txt 192.168.234.129 --exclude /path/to/rules.ignore -p ./backups/
   ```

## Логика (важно)

- Правила — **один набор на весь запуск**, применяются ко **всем** корням.
- Срабатывают по совпадению **в любом месте пути** (bare-имя `venv` матчит на любой
  глубине; `venv/` — только каталог; `/cache` — якорь к верхнему уровню корня).
- Фильтруется **рекурсивно раскрытое** содержимое. **Явно указанный в filelist
  путь до файла — обход исключений** (включается всегда: «указал → возьми»).
  Явный каталог — это корень, его потомки фильтруются.
- Совпавший каталог исключается целиком (внутрь не спускаемся).

## Ожидаемый результат (кратко)

- В `files/`/payload попадает дерево **без** исключённых путей.
- Исключённые точки (каталог/файл) записаны в manifest со статусом `excluded` и
  правилом-причиной (`copy.error = "excluded by rule: …"`); внутрь не
  перечисляется (нет раздувания на `node_modules`).
- В сводке — строка `Excluded: N`.
- Правила сохраняются в `session.json` (`exclude: […]`) → `diff` использует **те
  же** правила.

```
  Processed:        …
    Copied OK:      …
    Excluded:       3   (по правилам ignore)
```

## Справочно: ключи

| Ключ | По умолчанию | Описание |
|---|---|---|
| `--exclude [FILE]` | без флага — выкл.; со значением по умолчанию `./.syncignore` | подключить правила исключения из FILE (gitignore-стиль) |

- Флаг **не указан** → исключений нет (полное рекурсивное сохранение).
- `--exclude` без значения → `./.syncignore`; `--exclude FILE` → указанный файл.
- Файл не найден → ошибка.
- Действует и для локального режима: `... --local --no-sudo --exclude`.
