# UC4 — get → отредактировать → put (накат с правками)

`HOST1 → HOST_LOCAL1 → (правка) → HOST_LOCAL1* → HOST4`

Полный разбор с проверкой на хосте — в [../step_by_step.md](../step_by_step.md).

## Действия пользователя

1. **Забрать** (как [UC1](uc1-get.md)):
   ```bash
   python -m filesync get files.txt 192.168.234.129 --prefix clusterA --suffix message -p ./backups/
   ```
2. **Отредактировать локально** в папке сессии — только два места:
   - **контент** — править файлы прямо в `backups/<session>/files/...`;
   - **права / владелец / группа / выбор набора** — править `backups/<session>/restore.txt`
     (`<type> <perms> <owner> <group> <путь>`; `-` = «как в metadata»; удалить строку =
     не восстанавливать этот объект).
3. **Накатить** — та же команда, что и в [UC3](uc3-put.md) (никаких доп. ключей):
   ```bash
   SESS=./backups/192.168.234.129__<дата>__message
   python -m filesync put "$SESS" 192.168.234.129 --target-root /srv/restore --yes
   ```

## Ожидаемый результат (кратко)

`put` всегда читает `restore.txt` сессии и берёт контент из `files/`, поэтому
правки подхватываются автоматически: изменённый контент заливается, изменённые
атрибуты применяются, удалённые из `restore.txt` объекты не накатываются.
План показывает итоговые атрибуты; в конце — верификация и журнал.

Пример: у одного файла изменён контент, у другого — права, у третьего — группа,
у четвёртого — владелец; лишние объекты исключены правкой `restore.txt`;
`Restore: OK N/N`.

## Заметки

- Родительские каталоги get захватывает с реальными правами (есть в `restore.txt`);
  при накате на `/` уберите из `restore.txt` строки системных каталогов, которые
  не нужно трогать — они будут созданы нейтрально.
- Перезаливка контента сейчас затрагивает все выбранные файлы; точечная заливка
  только изменённых — [opt_attr_fast_path.md](../../todo/opt_attr_fast_path.md) (на будущее).
