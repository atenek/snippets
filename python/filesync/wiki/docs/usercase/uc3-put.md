# UC3 — put: накатить локальную копию на хост

`HOST_LOCAL1 → HOST3` (целевой хост может отличаться от исходного)

Единая форма: **`put <session> [HOST] --target-root <ROOT>`**. Источник данных —
всегда папка сессии: контент из `files/`, выбор объектов и атрибуты — из
`restore.txt`. `payload.tar` сам put не использует (это «холодный» архив).

## Действия пользователя

1. Иметь сессию (вывод get/diff).
2. Посмотреть план (ничего не пишет):
   ```bash
   python -m filesync put ./backups/192.168.234.129__<дата>__message 192.168.234.129 \
          --target-root /srv/restore --dry-run
   ```
3. Применить:
   ```bash
   python -m filesync put ./backups/192.168.234.129__<дата>__message 192.168.234.129 \
          --target-root /srv/restore --yes
   ```

Без правок `restore.txt`/`files/` это даёт «точно как было» (контент из `files/`,
атрибуты — фактические, предзаполненные get). Накат **с правками** — [UC4](uc4-get-modify-put.md).

## Роль `restore.txt`

- **Выбор:** восстанавливаются **только** объекты, чьи строки есть в `restore.txt`
  (удалите строки — эти пути не накатятся).
- **Атрибуты:** `<type> <perms> <owner> <group> <path>` (`type` = `d`/`f`/`l`,
  информативно); `-` = взять из metadata.
- Включает **родительские каталоги** (их get захватывает с реальными правами).
  Недостающие родители, которых нет в restore.txt, создаются `mkdir -p`
  нейтрально (без смены атрибутов — системные каталоги не клоббятся).

## Ожидаемый результат (кратко)

Объекты воссоздаются под `--target-root` с владельцем/правами/временем/симлинками.
Затем верификация на хосте (`stat`+`md5sum`) и журнал `..._restore/restore.json|.csv` (RO).
Консоль: `OK / skipped / attr mismatch / content mismatch / failed`. Код `0`/`2`.

> Устройства/FIFO/сокеты через put не восстанавливаются — только вручную из `payload.tar`.

## Справочно: ключи и значения по умолчанию

| Ключ | По умолчанию | Описание |
|---|---|---|
| `session` (поз.) | — (обязателен) | папка сессии (`files/` + `restore.txt` + `metadata/`) |
| `hostname` (поз.) | из `session.json` | целевой хост (здесь `192.168.234.129`) |
| `--target-root` | — (**обязателен**) | корень назначения (`/` или `/srv/restore`) |
| `--dry-run` | off | показать план без записи |
| `--yes` | off (в не-tty без него — отказ; в tty — запрос) | подтвердить применение |
| `--backup-existing` | off | сохранить перезаписываемые цели до изменения |
| `--no-set-times` | off (mtime ставится) | не восстанавливать mtime |

Выбор подмножества и атрибуты задаются через `restore.txt` (отдельных ключей
`--apply-list`/`--from-files`/`--only` нет — это и есть упрощение).

Безопасность: `--target-root` обязателен; `/` — только с `--yes`; защита от path-traversal.
