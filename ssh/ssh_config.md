Опция `-F` задаёт файл конфигурации SSH.
Если указать `/dev/null`, то пользовательский файл `~/.ssh/config` не будет прочитан.

```bash
ssh -F /dev/null user@host
```

Если требется игнорировать системный конфиг `/etc/ssh/ssh_config`, используйте:

```bash
ssh -F /dev/null \
    -o GlobalKnownHostsFile=/dev/null \
    -o UserKnownHostsFile=/dev/null \
    user@host
```

Но это отключает только файлы known_hosts.
Полностью отключить чтение системного `/etc/ssh/ssh_config` штатной опцией нельзя!
OpenSSH всегда читает системный конфиг, а затем указанный через `-F` пользовательский конфиг.

Проверить итоговые параметры, которые реально будут применены:

```bash
ssh -G -F /dev/null user@host
```

Команда выведет эффективную конфигурацию после обработки всех настроек.
Для отладки, если есть подозрение, что какие-то параметры подхватываются из конфигов.

Для указания группы с одинаковыми параметрами можно использовать шаблоны (*, ?) в имени хоста.

Например, для всех хостов из диапазона 192.168.1.*:

Host 192.168.1.*
    User alex
    Port 2222
    IdentityFile ~/.ssh/id_ed25519

Для нескольких диапазонов:

Host 192.168.1.* 192.168.2.*
    User alex
    Port 2222

Можно использовать ? для одного символа:

Host server?
    User alex

Подойдёт для server1, server2, ..., server9.

Host — это шаблон или псевдоним, который вы указываете в команде ssh.
HostName — это реальный адрес или DNS-имя, к которому SSH будет подключаться.
Для групп IP-адресов HostName обычно не нужен вообще.
