Схема будет вертикальной — три яруса (TRex, средний ярус БН+DecapHost, и петля возврата) со всеми четырьмя направлениями трафика подписанными отдельно.Теперь все четыре направления отражены явно и корректно.

**Три исправления относительно предыдущей схемы:**

Первое — связь Port 1 rx теперь подписана правильно: входящий трафик на этот порт это чистый UDP после декапсуляции на DecapHost, а не IPIP.

Второе — добавлена связь Port 0 rx, которой раньше не было вовсе. Ответный трафик (DSR) приходит именно сюда — на тот же порт, с которого уходили запросы.

Третье — DSR-петля (Port 1 tx → Port 0 rx) показана отдельной пунктирной линией, которая физически не проходит через LB и DecapHost — она идёт напрямую, в обход обоих промежуточных компонентов. Это и есть суть Direct Server Return.
