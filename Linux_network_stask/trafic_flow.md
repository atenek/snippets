# Обработка сетевого трафика в Linux 

Это цепочка шагов внутри ядра (kernel), где пакеты проходят через сетевой стек.
Шаги по порядку, от прихода пакета до его обработки приложением.

---

# 📥 1. Приход пакета на сетевую карту (NIC)

Когда пакет приходит:

1. Сетевая карта (NIC) принимает сигнал
2. Записывает пакет в буфер (DMA → RAM)
3. Генерирует **прерывание (interrupt)** или использует **NAPI (polling)**

Современные системы чаще используют **NAPI**, чтобы избежать перегрузки от большого количества прерываний.

---

# ⚙️ 2. Драйвер сетевой карты

Драйвер:

* забирает пакет из буфера
* оборачивает его в структуру ядра **`sk_buff` (socket buffer)**

`sk_buff` — это основной объект, представляющий пакет внутри Linux.

---

# 🌐 3. Попадание в сетевой стек (Kernel Network Stack)

Дальше пакет проходит уровни:

## 📦 L2 — Канальный уровень (Ethernet)

* проверка MAC-адреса
* удаление Ethernet-заголовка

Если пакет не для нас → может быть отброшен

---

## 🌍 L3 — IP уровень

* проверка IP-адреса назначения
* фрагментация/дефрагментация
* маршрутизация (routing decision)

👉 Здесь решается:

* пакет для локальной машины?
* или его надо переслать дальше (forwarding)?

---

## 🚦 Netfilter (iptables / nftables)

На этом этапе пакет проходит через хуки:

* PREROUTING
* INPUT
* FORWARD
* OUTPUT
* POSTROUTING

Здесь возможны:

* firewall (drop/accept)
* NAT (маскарадинг)
* логирование

---

## 🚚 L4 — TCP/UDP

* TCP: проверка состояния соединения, сбор сегментов
* UDP: просто передача данных

---

# 🧠 4. Доставка в сокет (Socket Layer)

Ядро:

* находит нужный сокет (по IP + портам)
* кладёт данные в очередь сокета

---

# 👨‍💻 5. Передача приложению

Приложение:

* читает данные через:

  * `recv()`
  * `read()`

👉 Если используется:

* **epoll / select** — приложение уведомляется о готовности данных

---

# 📤 Обратный путь (исходящий трафик)

Когда приложение отправляет данные:

1. `send()` → данные попадают в сокет
2. TCP/UDP формирует пакет
3. IP добавляет заголовок
4. Netfilter (OUTPUT, POSTROUTING)
5. драйвер → NIC → сеть

---

# ⚡ Важные механизмы оптимизации

## 🔹 NAPI

* уменьшает количество прерываний
* использует polling при высокой нагрузке

## 🔹 GRO / GSO

* объединение пакетов (оптимизация CPU)

## 🔹 RSS (Receive Side Scaling)

* распределение пакетов по CPU

## 🔹 XDP / eBPF

* обработка пакетов на очень ранней стадии (ещё до стека)
* используется для high-performance (например, DDoS фильтрация)

---

# 🧩 Краткая схема

```
NIC → Driver → sk_buff → L2 → L3 → Netfilter → L4 → Socket → Application
```

---

# 💡 Итог

Linux обрабатывает пакеты полностью в ядре через сетевой стек, проходя уровни OSI и различные фильтры. Ключевые элементы:

* `sk_buff` — представление пакета
* netfilter — контроль/фильтрация
* socket layer — связь с приложением
