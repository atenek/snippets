Добавить **отладочную информацию (debug symbols)** при компиляции через `gcc`, используется флаг `-g`.

---

## Базовый вариант

```bash
gcc -g test.c -o test
```

Это добавит в бинарник информацию для отладчиков (gdb, lldb и т.д.).

---

## Полезные комбинации

### 1. С оптимизациями (реалистичный сценарий)

```bash
gcc -g -O2 test.c -o test
```

* `-g` — отладочная информация
* `-O2` — оптимизация
  ⚠️ Но имей в виду: оптимизации могут «ломать» удобство отладки (переменные исчезают, код переупорядочен)

---

### 2. Для максимально удобной отладки

```bash
gcc -g -O0 test.c -o test
```

* `-O0` — без оптимизаций (лучше для gdb)

---

### 3. С предупреждениями (рекомендуется)

```bash
gcc -g -O0 -Wall -Wextra test.c -o test
```

---

## Проверка, что debug symbols есть

```bash
file test
```

Должно быть что-то вроде:

```
... with debug_info ...
```

Или:

```bash
readelf -S test | grep debug
```

---

## Использование с core dump

После падения:

```bash
gdb ./test core
```

Если символы есть — увидишь:

* имена функций
* строки кода
* переменные

Без `-g` — только адреса и дизассемблированный код.

---

## Отдельно: уменьшение размера (опционально)

Можно вынести debug-информацию в отдельный файл:

```bash
gcc -g test.c -o test
objcopy --only-keep-debug test test.debug
strip --strip-debug test
objcopy --add-gnu-debuglink=test.debug test
```
